## Design Context

### Users

BatchImager is for people producing many ecommerce product images from rough source photos, such as operators handling warehouse-shot flower photos. They need to scan, batch-process, and refine images quickly without waiting on one image before moving to the next.

### Brand Personality

Quiet, precise, efficient.

### Aesthetic Direction

The interface should feel like a compact macOS/Windows productivity tool, not a marketing page or web dashboard. Images are the primary surface. Controls should be small, Chinese-first, and restrained.

### Design Principles

- Let images occupy as much space as possible.
- Separate image cells with hairline dividers instead of card chrome.
- Put file names, status, and lightweight actions inside the image area.
- Use icon-only status marks in image corners.
- Keep generation features honest: show planned entry points, but do not fake working AI generation before the service exists.
